# MES系统 — 前端UI布局规范

> 版本：v1.1  
> 生效日期：2026-06-29  
> 状态：**已生效**

## 1. 总则

本规范定义了 MES 系统（制造执行系统）所有列表页面的 UI 布局标准，确保全项目界面风格一致、用户体验统一。**所有开发人员在新页面开发或旧页面重构时，必须严格遵守本规范。**

---

## 1.1 单据布局一致性规范（新增 2026-06-29）

### 规则1：字段布局一致性

**同一张单据，创建/编辑/查看三种状态的字段位置/顺序/分组必须保持一致。**

| 规则ID | 规则描述 | 强制等级 | 适用范围 |
|--------|---------|----------|----------|
| **UI-FLD-01** | 创建表单的字段位置/顺序/分组，必须与查看详情一致 | **MUST** | 业务单据 + 主数据 |
| **UI-FLD-02** | 编辑表单的字段位置/顺序/分组，必须与查看详情一致 | **MUST** | 业务单据 + 主数据 |
| **UI-FLD-03** | 查看详情时行项目列的顺序/显示逻辑，必须与创建表单一致（如动态列切换） | **MUST** | 业务单据 |

**错误示例**：
- ❌ 创建时表头字段顺序是「类型→编号→部门→采购组」，查看时变成「类型→编号→日期→工厂→部门→采购组」
- ❌ 创建时 Z01 模式行项目显示「科目分配类别」列，查看时该列消失

**正确示例**：
- ✅ 创建和查看的表头字段顺序完全一致
- ✅ 查看时行项目列根据单据类型动态切换，与创建时逻辑一致

---

### 规则2：页签结构一致性

**同一张单据，创建/编辑时的页签结构，与查看时的页签结构必须一致。**

| 规则ID | 规则描述 | 强制等级 | 适用范围 |
|--------|---------|----------|----------|
| **UI-TAB-01** | 创建/编辑时有页签的，查看详情必须有相同页签 | **MUST** | 业务单据 + 主数据 |
| **UI-TAB-02** | 页签名称和顺序必须一致（如创建时是「抬头→工序→物料」，查看时也必须相同） | **MUST** | 业务单据 + 主数据 |
| **UI-TAB-03** | 创建/编辑时平铺一页的，查看详情也必须平铺一页（不得新增页签） | **MUST** | 业务单据 + 主数据 |

**错误示例**：
- ❌ 创建工单时分页签填写（抬头、工序、物料），查看时所有字段平铺在一页
- ❌ 创建时平铺，查看时反而分页签

**正确示例**：
- ✅ 创建/编辑/查看都有相同的页签结构
- ✅ 创建/编辑/查看都是平铺布局（无页签）

---

### 规则3：弹窗尺寸一致性

**同一张单据，创建/编辑/查看时的弹窗尺寸必须一致。**

| 规则ID | 规则描述 | 强制等级 | 适用范围 |
|--------|---------|----------|----------|
| **UI-MOD-01** | 同一张单据的创建弹窗、编辑弹窗、查看弹窗，必须使用相同的弹窗尺寸（如都是 `modal-lg` 或都是 `modal-xl`） | **MUST** | 业务单据 + 主数据 |
| **UI-MOD-02** | 弹窗尺寸在 `showModal()` 调用时通过第三个参数指定（如 `'modal-lg'`） | SHOULD | 业务单据 + 主数据 |

**错误示例**：
- ❌ 创建采购申请用 `modal-xl`，查看采购申请用 `modal-lg`
- ❌ 编辑设备主数据用 `modal-lg`，查看设备主数据用默认尺寸

**正确示例**：
- ✅ 同一张单据的所有弹窗都使用 `modal-lg`
- ✅ 同一张单据的所有弹窗都使用 `modal-xl`

## 2. 列表页标准布局结构（从上到下）

```
┌─────────────────────────────────────────────┐
│  ① 页面标题栏（Header）                     │  flex-shrink:0
│     标题 + 副标题 | 操作按钮                │
├─────────────────────────────────────────────┤
│  ② 统计卡片区（可选）                        │  flex-shrink:0
│     [总数] [状态A] [状态B] ...              │
├─────────────────────────────────────────────┤
│  ③ 筛选工具栏（Filter Bar）                  │  flex-shrink:0
│     [筛选字段] ... [查询] [重置]            │
├─────────────────────────────────────────────┤
│  ④ 数据表格区（Table Wrapper）               │  flex:1（占据剩余空间）
│  ┌───────────────────────────────────┐      │
│  │ <thead> 表头                      │      │
│  │ <tbody> 数据行                    │      │
│  └───────────────────────────────────┘      │
├─────────────────────────────────────────────┤
│  ⑤ ★ 分页控件栏（Pagination Toolbar） ★    │  flex-shrink:0
│     [共 N 条]       [‹] 第x/y页 [›] [每页] │
└─────────────────────────────────────────────┘
```

### 核心规则

| 规则ID | 规则描述 | 强制等级 |
|--------|---------|----------|
| **UI-PAG-01** | **分页控件必须置于数据表格/内容区域的正下方** | **MUST** |
| **UI-PAG-02** | 分页控件不得置于表格上方或侧边 | **MUST NOT** |
| **UI-PAG-03** | 分页栏使用 `class="list-toolbar"` + `style="flex-shrink:0;"` | SHOULD |
| **UI-PAG-04** | 分页栏包含：左侧记录数统计 + 右侧翻页控件 | SHOULD |
| **UI-LAY-01** | 外层容器使用 `display:flex; flex-direction:column; height:calc(100vh - 56px);` | MUST |
| **UI-LAY-02** | 表格容器使用 `flex:1` 自动填充剩余空间 | MUST |

## 3. DOM 结构模板（参考实现）

```html
<div style="display:flex;flex-direction:column;height:calc(100vh - 56px);">
  <!-- ① 标题栏 -->
  <div style="...flex-shrink:0;">
    <h1>页面标题</h1>
    <button>+ 操作</button>
  </div>

  <!-- ② 统计卡片（可选） -->
  <div style="...flex-shrink:0;">
    <!-- stat-card 组件 -->
  </div>

  <!-- ③ 筛选栏 -->
  <div class="filter-bar" style="flex-shrink:0;">
    <input /> <select />
    <button onclick="search()">查询</button>
  </div>

  <!-- ④ 表格区域（flex:1 占满剩余高度） -->
  <div class="table-wrapper" style="flex:1;">
    <table class="data-table">
      <thead><tr>...</tr></thead>
      <tbody id="xxxTableBody"></tbody>
    </table>
  </div>

  <!-- ⑤ ★ 分页控件（必须位于此处：表格之后）★ -->
  <div class="list-toolbar" style="flex-shrink:0;">
    <div class="list-info">
      <span class="list-count" id="xxxCount">共 N 条</span>
    </div>
    <div class="pagination">
      <button class="pagination-btn" id="xxxPrev" disabled onclick="Xxx.prevPage()">‹</button>
      <span class="pagination-info" id="xxxPageInfo">第 1 / N 页</span>
      <button class="pagination-btn" id="xxxNext" onclick="Xxx.nextPage()">›</button>
      <select class="page-size-select" onchange="Xxx.changePageSize()">
        <option value="10">10条</option>
        <option value="20">20条</option>
        <option value="50">50条</option>
      </select>
    </div>
  </div>
</div>
```

## 6. 设计依据

1. **用户视觉习惯**：用户从上到下浏览数据后，翻页控件自然出现在视线末端，符合 F 型阅读模式。
2. **行业惯例**：主流企业管理系统（SAP、钉钉、飞书、企业微信）均将分页置于数据列表底部。
3. **空间利用效率**：顶部空间留给筛选条件和操作按钮，底部固定分页不占用数据展示区域。
