## 用户需求

用户要求所有弹窗（modal/dialog）尽量大一点，不要留滚动条，拖来拖去很烦。

## 当前问题

1. 默认弹窗宽度只有 `620px`，绝大多数弹窗未指定尺寸类，导致内容非常拥挤
2. `.modal-body` 设置了 `max-height: calc(90vh - 148px)` 和 `overflow-y: auto`，内容稍多就出现滚动条
3. 全项目约 108 处 `showModal()` 调用中，只有 12 处使用了 `modal-lg` 或 `modal-xl`，其余均使用默认 620px 窄宽度
4. 弹窗高度 `max-height: 90vh` 偏保守，还有 10vh 的空闲屏幕空间未利用

## 修改目标

1. 增大弹窗默认宽度，让表单和详情内容尽量在一屏内显示
2. 增大弹窗最大高度，充分利用屏幕空间
3. 调整弹窗尺寸变体的宽度，让大尺寸弹窗真正大
4. 确认类弹窗（删除确认、操作确认等简短提示）保持较小尺寸，避免过大

### 具体 CSS 修改（V3/css/common.css）

#### 1. 默认弹窗尺寸（第 121-126 行）

```css
/* 修改前 */
.modal {
  max-width: 620px;
  max-height: 90vh;
}
/* 修改后 */
.modal {
  max-width: 1400px;   /* 增大默认宽度，大多数弹窗直接受益 */
  max-height: 96vh;    /* 充分利用屏幕高度 */
}
```

#### 2. 弹窗尺寸变体（第 129-136 行）

```css
/* 修改前 */
.modal-sm  { max-width: 460px; }
.modal-md  { max-width: 680px; }
.modal-lg  { max-width: 820px; }
.modal-xl  { max-width: 1020px; }
.modal-xxl { max-width: 1340px; }
.mp-modal  { max-width: 1120px; max-height: 88vh; }

/* 修改后 */
.modal-sm  { max-width: 480px; }   /* 确认弹窗用，略增大 */
.modal-md  { max-width: 900px; }   /* 中等弹窗 */
.modal-lg  { max-width: 1400px; }  /* 与默认一致 */
.modal-xl  { max-width: 1700px; }  /* 大弹窗，接近全屏 */
.modal-xxl { max-width: 96vw; }    /* 超大弹窗，几乎全屏 */
.mp-modal  { max-width: 1500px; max-height: 94vh; }
```

#### 3. 弹窗内容区域高度（第 153-157 行）

```css
/* 修改前 */
.modal-body {
  max-height: calc(90vh - 148px);
}
/* 修改后 */
.modal-body {
  max-height: calc(96vh - 148px);  /* 增加可用高度 */
}
```

#### 4. split-wrap 弹窗高度（第 169 行）

```css
/* 修改前 */
.modal-body:has(.split-wrap) { max-height: calc(90vh - 108px); }
/* 修改后 */
.modal-body:has(.split-wrap) { max-height: calc(96vh - 108px); }
```

#### 5. 移动端 480px 断点（第 386-398 行）

```css
/* 修改前 */
@media (max-width: 480px) {
  .modal { max-height: 92vh; }
  .modal-body { max-height: calc(92vh - 140px); }
}
/* 修改后 */
@media (max-width: 480px) {
  .modal { max-height: 96vh; }
  .modal-body { max-height: calc(96vh - 140px); }
}
```

### JS 文件修改原则

- 只有简短确认文字（无表单、无列表）的弹窗才加 `modal-sm`
- 包含表单、详情内容、数据表格的弹窗不加尺寸类（使用新的默认 1400px 宽度）
- `showModal(title, body, btns, 'modal-lg')` 类调用保持原样（CSS 修改后 `modal-lg` = 1400px，与默认一致，但显式声明有助于代码可读性）

### 不需要修改的文件

- 已使用 `modal-xl` 的弹窗（如 maintenance-tasklist.js）：CSS 修改后 `modal-xl` 会更大（1700px），无需改动
- 已使用 `modal-lg` 的弹窗：CSS 修改后 `modal-lg` = 1400px，与新的默认宽度一致，无需改动

## 实施顺序

1. 先改 CSS，让所有未指定尺寸类的弹窗立即变大
2. 再改 JS 确认类弹窗，加上 `modal-sm` 避免确认弹窗过大
