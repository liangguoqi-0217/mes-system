## 用户需求

### 1. 物料类型值域修正

物料类型的值域应为：Z001-成品、Z002-半成品、Z003-原料、Z004-辅料、Z005-包材、Z006-耗材及其他。此字段与物料主数据存储在同一个表中（即 `materialData` 中的 `materialType` 字段）。

**当前问题**：`safety-stock-alert.js` 中的 `_getMatType()` 方法是根据物料号前缀硬编码判断的，且下拉框选项是备品备件分类（600~608），不是正确的物料类型值域。

**修正方案**：

- `safety-stock-alert.js` 的 `_getMatType()` 改为从 `materialData` 中查找对应 `code === matCode` 的记录，读取其 `materialType` 和 `materialTypeName`
- 查询条件中的「物料类型」下拉框选项改为：Z001-成品、Z002-半成品、Z003-原料、Z004-辅料、Z005-包材、Z006-耗材及其他
- 同步更新 `data.js` 中的 `materialTypeOptions` 常量（当前值是 Z001-原材料、Z002-包装材料等，需要更正）

### 2. 页面顶部渐变色统一为蓝色

**当前问题**：`safety-stock-alert.js` 第39行页面顶部渐变色是黄色（`#d97706,#f59e0b`），其他所有页面均使用蓝色（`var(--primary),var(--primary-light)`）。

**修正方案**：

- 将 `safety-stock-alert.js` 第39行的渐变色改为 `linear-gradient(135deg,var(--primary),var(--primary-light))`
- 将此规则写入框架规范（在 `SOUL.md` 或类似文档中记录：所有页面顶部渐变色统一使用 `var(--primary)` 蓝色）

### 3. 准备库存不足的 Demo 数据（红灯数据）

**当前问题**：需要有一些库存低于安全库存的物料，在报表中显示红灯状态。

**修正方案**：

- 在 `sparePartsStockData` 中新增或调整部分物料的库存数据，使其 `unrestrictedQty + qualityQty < safetyStock`，触发红灯
- 同步调整 `_getSafetyStock()` 中对应物料的安全库存值，确保红灯数据合理

### 4. 采购申请新增第三种创建方式

**需求变更**：推翻之前「在安全库存预警页面点击创建采购申请」的设计，改为在采购申请页面新增第三种创建方式：「按安全库存预警创建」。

**交互流程**：

1. 用户点击采购申请页面的「新建申请」按钮
2. 弹出「选择创建方式」弹窗，原有「手工填写」和「模板批导」两种卡片，新增第三种：「按安全库存预警创建」
3. 用户点击「按安全库存预警创建」卡片
4. 弹窗关闭，打开一个新的弹窗/面板，内部加载安全库存预警数据（复用 `_buildAlertData()` 逻辑）
5. 用户勾选红灯物料（仅红灯可勾选）
6. 点击「确认创建」，将勾选的物料带入采购申请创建表单（Z01 模式，自动填充物料号、短文本、申请数量=缺货数量等）

**修正方案**：

- 修改 `sp-purchase.js` 的 `openNewModal()` 方法，在弹窗中新增第三个卡片：「按安全库存预警创建」
- 新增 `openAlertCreateModal()` 方法，打开安全库存预警数据弹窗，内部显示物料列表、勾选框、确认按钮
- 新增 `_fillFromAlertSelect(lines)` 方法，将勾选的物料数据填入采购申请表单

_getMatTypeFromMaterialData(matCode) {
const m = materialData.find(x => x.code === matCode && x.deleteFlag !== 'D');
if (m) return (m.materialType || '') + '-' + (m.materialTypeName || '');
return '';
}
