\# TABLE GUIDELINE



> Table Design Guideline



Version: 1.0



\---



\# 1. Purpose（目的）



表格用于展示业务数据。



而不是展示数据库。



优秀的表格，应帮助用户快速发现信息、完成业务，而不是一次性展示所有内容。



\---



\# 2. Core Principle（核心原则）



表格首先服务于业务。



其次服务于数据。



默认满足大多数用户。



同时允许适度个性化配置。



\---



\# 3. Default Sort（默认排序）



默认采用：



更新时间倒序。



让用户优先关注最新变化的数据。



特殊业务可根据业务需求调整排序方式。



\---



\# 4. First Column（第一列）



第一列默认展示业务主标识。



例如：



订单号。



设备编号。



物料编码。



避免默认使用勾选框作为第一列。



\---



\# 5. Column Display（列展示）



默认展示完整业务信息。



同时允许用户：



\- 显示列

\- 隐藏列

\- 调整列顺序（可选）



避免因为字段较多而直接删除业务信息。



\---



\# 6. Operation Column（操作列）



操作统一放置于最后一列。



常见操作包括：



\- 查看

\- 编辑

\- 删除（逻辑删除）

\- 更多



进入详情应通过明确操作触发。



避免依赖双击等隐式交互。



\---



\# 7. Status Display（状态展示）



状态统一采用：



颜色 + 文字。



颜色用于快速识别。



文字用于准确表达。



避免仅依赖颜色传达状态。



\---



\# 8. Data Alignment（数据对齐）



默认保持统一左对齐。



包括：



文本。



金额。



日期。



特殊场景可根据业务调整。



整体应保持一致。



\---



\# 9. Table Height（表格高度）



表格默认占据页面剩余空间。



充分利用可视区域。



减少滚动切换。



\---



\# 10. Pagination（分页）



默认每页展示：



100 条数据。



支持用户自行调整分页数量。



减少高频业务中的翻页成本。



\---



\# 11. Empty State（空状态）



无数据时统一展示：



暂无数据。



保持页面简洁。



避免无意义装饰。



\---



\# 12. Loading（加载状态）



数据加载期间统一采用：



Loading。



保持用户明确感知系统状态。



\---



\# 13. Large Tables（大型表格）



设计阶段应避免出现过多字段。



当业务确实需要展示较多字段时：



默认全部展示。



同时允许用户自定义显示列。



避免为了减少字段而损失业务信息。



\---



\# 14. Review Checklist（评审清单）



设计完成后，应确认：



□ 默认排序是否符合业务。



□ 第一列是否为业务主标识。



□ 是否支持用户配置显示列。



□ 操作是否集中管理。



□ 状态是否采用颜色 + 文字。



□ 表格是否充分利用页面空间。



□ 是否避免隐式操作。



优秀的表格，不是数据列表。



而是用户完成业务工作的主要界面。
